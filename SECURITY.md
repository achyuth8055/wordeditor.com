# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities. 

Instead, please report security vulnerabilities to **security@wordeditor.online** with:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if available)

We appreciate your responsible disclosure and will:
1. Acknowledge receipt within 24 hours
2. Provide an update on progress within 5 business days
3. Credit you in the fix release (if desired)

## Security Measures

- HTTPS-only communication
- No sensitive data stored locally
- Regular dependency updates
- Security headers configured
- XSS and CSRF protection
- Input validation and sanitization
- API rate limiting
- DDoS protection via CDN

## Privacy Practices

- No personal data collection
- No third-party tracking
- GDPR compliant
- Cookie-free (except analytics with consent)
- Open data policy

Thank you for helping keep WordEditor.online secure!
